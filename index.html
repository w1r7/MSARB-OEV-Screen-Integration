DM1 - Active Diagnostic Message

How DM1 works - expand for explanation

DM1 is the active fault message. It uses PGN 65226 / 0x00FECA.

SPN identifies what signal or component has the fault. FMI identifies how it failed. OC is the occurrence count.

For this customer integration, the calculator uses single-packet DM1 only. If several faults are active, the shared trace rotates through them one-by-one. No DM1 BAM / multi-packet message is generated.

DM1 Simple View - Fault Selector

How to use this view

Choose a mapped fault from the list. The tool shows the DM1 message that MAX-SAFE would transmit.

SPN means the faulty component or signal. FMI means how it failed. For this customer integration, DM1 is sent as single-packet only. If several faults are active, the trace rotates through them one-by-one. When no faults are active, MAX-SAFE sends the no-fault DM1 frame.

Make, Model, Serial Number, and Unit Number are optional in the standard message, but the ASCII * delimiter is always required. In this calculator, Serial Number is mandatory because this tab is used to test the MAX-SAFE serial number response.

Common CAN Trace



How the shared trace works - expand for explanation

The trace area represents one shared CAN bus log for the whole calculator. Every tab writes to the same trace buffer and the same export file.


Start begins trace generation. Use the filters to show or hide each message.


Use the Trace filters to include or exclude each cyclic message from the trace. If a CM/MSB message is selected but not activated in its tab, the trace transmits FF FF FF FF FF FF FF FF.


Customer requests still simulate external J1939 requests. Software and Serial append one response into the same trace.
